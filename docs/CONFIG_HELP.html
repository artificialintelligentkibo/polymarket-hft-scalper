Platform Configuration Helper

Polymarket HFT Scalper is now a multi-strategy platform.

Use this page as a fast mental map for the current control model: runtime modes, ACTIVE_STRATEGY presets, strategy engines, dashboards, and reporting. For exact defaults and validation ranges, the code in src/config.ts is still the source of truth.

Runtime Mode

Simulation SIMULATION_MODE=true and DRY_RUN=true
Product Test PRODUCT_TEST_MODE=true with real but constrained flow
Production Live trading with normal runtime controls
Step 2

Preset Layer

ACTIVE_STRATEGY is the high-level switch. It can turn engines on or off after your raw env has already been parsed.

Important: CURRENT_SNIPER means "no preset override", not "only sniper".
Step 3

Engine Layer

After preset overrides, individual engines such as OBI_ENGINE_ENABLED, SNIPER_MODE_ENABLED, and VS_ENGINE_ENABLED decide what actually runs.

Strategy Presets

Preset Best Use What It Does
CURRENT_SNIPER Manual compatibility mode Leaves your flags as configured.
ORDER_BOOK_IMBALANCE Modern OBI-first live deployment Turns on OBI, turns off sniper/latency/paired arb, disables regular MM, keeps Binance feed only when needed for OBI or VS.
PAIRED_ARBITRAGE Lowest-directionality deployment Runs paired arb only and disables directional layers.
ALL True multi-strategy platform mode Runs OBI and sniper together, preserves VS if enabled, and pairs well with LOTTERY_LAYER_ENABLED=true when you want follow-on convexity. Regular MM still stays off.
Why regular MM turns off in OBI and ALL: OBI and VS already have their own quote paths. Running the generic MM_QUOTE engine in parallel would create confusing same-market ownership conflicts.

Engine Map

OBI

Standalone order-book imbalance trading with its own entries, exits, and maker follow-ons.

  • OBI_ENGINE_ENABLED
  • OBI_SHADOW_MODE
  • OBI_BINANCE_REQUIRE_ALIGNMENT
  • OBI_TIME_TAKE_PROFIT_MS

VS Engine

Binance latency arb with slot-open strike, realized volatility, fair-value MM, and late momentum.

  • VS_ENGINE_ENABLED
  • VS_SHADOW_MODE
  • VS_MOMENTUM_THRESHOLD_SIGMAS
  • VS_TARGET_EXIT_PRICE

Sniper

Fast Binance-led directional entry engine with PM-lag and fee-aware filters.

  • SNIPER_MODE_ENABLED
  • SNIPER_MIN_BINANCE_MOVE_PCT
  • SNIPER_MIN_EDGE_AFTER_FEES
  • REGIME_FILTER_ENABLED

Regular MM

Passive quoting from the dedicated quoting engine. Separate from OBI and VS quoting.

  • MARKET_MAKER_MODE
  • DYNAMIC_QUOTING_ENABLED
  • MM_QUOTE_SHARES
  • MM_MAX_GROSS_EXPOSURE_USD

Paired Arb

Parity-based YES plus NO trading with atomic pair handling and async starter-leg support.

  • PAIRED_ARB_ENABLED
  • PAIRED_ARB_ASYNC_ENABLED
  • PAIRED_ARB_MIN_NET_EDGE
  • PAIRED_ARB_MAX_PAIR_COST

Lottery

Small fixed-risk opposite-side convexity layer, usually following a trigger fill.

  • LOTTERY_LAYER_ENABLED
  • LOTTERY_MAX_RISK_USDC
  • LOTTERY_ONLY_AFTER_SNIPER
  • LOTTERY_MAX_PER_SLOT
Do not confuse these: ORDER_BOOK_IMBALANCE_ENABLED is only a filter for regular MM activation. OBI_ENGINE_ENABLED is the real OBI strategy.

Shared Controls

Group Examples Why It Matters
Shared risk GLOBAL_MAX_EXPOSURE_USD, MAX_DRAWDOWN_USDC, MAX_NET_YES, MAX_NET_NO All engines feed into the same bankroll and directional limits.
Execution ORDER_TYPE, ORDER_TYPE_FALLBACK, POST_ONLY_ONLY, SLIPPAGE_TOLERANCE Changes how every layer reaches the book.
Compounding COMPOUNDING_ENABLED, COMPOUNDING_GLOBAL_EXPOSURE_PCT Moves the platform from static sizing to bankroll-aware sizing.
Monitoring STATUS_CHECK_INTERVAL_MS, AUTO_PAUSE_ON_INCIDENT Lets the runtime auto-pause entries on platform incidents.

Dashboards And Runtime Files

Terminal Dashboard

Launch with npm run scalper -- dashboard.

  • Active markets
  • Open positions
  • OBI session and gate breakdown
  • VS decisions
  • Layer status and recent signals

HTTP Dashboard

Enable with:

  • DASHBOARD_ENABLED=true
  • DASHBOARD_HOST=0.0.0.0
  • DASHBOARD_PORT=3847

Runtime Status

The dashboards read reports/runtime-status.json, which now includes:

  • layer status
  • portfolio, cash, available balance
  • OBI stats
  • VS stats
  • recent signals and skipped signals
